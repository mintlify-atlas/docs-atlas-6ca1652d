HuelvaPedia uses Firebase Realtime Database as its cloud-based backend to store and serve all encyclopedia content. This guide walks you through creating a Firebase project, configuring it for Android, and setting up the database structure.

<Warning>
  **Security Notice:** The `google-services.json` file is excluded from version control (listed in `.gitignore`) because it contains project-specific credentials. Each developer must create their own Firebase project and configuration file.
</Warning>

## Why Firebase?

HuelvaPedia leverages Firebase for several key advantages:

- **Cloud-Based Storage** - No local database; all content fetched dynamically
- **Real-Time Updates** - Content updates without requiring app updates
- **Lightweight App** - Reduces app size by keeping data in the cloud
- **Easy Content Management** - Update information via Firebase Console
- **Scalability** - Handles growing content seamlessly

## Creating a Firebase Project

Start by creating a new Firebase project in the Firebase Console.

<Steps>
  <Step title="Access Firebase Console">
    Navigate to [Firebase Console](https://console.firebase.google.com/) and sign in with your Google account.
  </Step>
  
  <Step title="Create New Project">
    Click **Add project** or **Create a project**.
    
    Enter a project name (e.g., "HuelvaPedia Dev" or "HuelvaPedia").
  </Step>
  
  <Step title="Configure Google Analytics (Optional)">
    You can enable or disable Google Analytics for this project. For development purposes, it's optional.
    
    Click **Continue** after making your selection.
  </Step>
  
  <Step title="Complete Project Creation">
    If you enabled Analytics, select or create an Analytics account.
    
    Click **Create project** and wait for the project to be provisioned.
  </Step>
  
  <Step title="Navigate to Project Overview">
    Once complete, click **Continue** to enter your Firebase project dashboard.
  </Step>
</Steps>

## Adding Android App to Firebase

Now register your Android app with the Firebase project.

<Steps>
  <Step title="Start Android App Registration">
    On the Project Overview page, click the **Android** icon to add an Android app.
  </Step>
  
  <Step title="Enter Package Name">
    In the "Register app" step, enter the Android package name:
    
    ```
    com.example.huelvapedia
    ```
    
    <Warning>
      **Critical:** This package name must match exactly with the `applicationId` in `app/build.gradle` and the `namespace` in `AndroidManifest.xml`. Do not modify it.
    </Warning>
  </Step>
  
  <Step title="Add App Nickname (Optional)">
    Optionally provide a nickname like "HuelvaPedia Android" to identify this app in the console.
  </Step>
  
  <Step title="Skip Debug Signing Certificate">
    The debug signing certificate SHA-1 is optional for Realtime Database. You can leave it blank and click **Register app**.
  </Step>
</Steps>

## Downloading google-services.json

The `google-services.json` file contains configuration and API keys for your Firebase project.

<Steps>
  <Step title="Download Configuration File">
    After registering the app, Firebase will prompt you to download `google-services.json`.
    
    Click **Download google-services.json** to save the file to your computer.
  </Step>
  
  <Step title="Locate Your Project's app/ Directory">
    In your file explorer, navigate to your cloned HuelvaPedia project:
    
    ```
    HuelvaPedia/app/
    ```
  </Step>
  
  <Step title="Place the File">
    Move or copy the downloaded `google-services.json` file into the `app/` directory:
    
    ```plaintext
    HuelvaPedia/
    └── app/
        ├── build.gradle
        └── google-services.json  ← Place here
    ```
    
    <Info>
      The file must be placed directly in the `app/` folder, not in subdirectories like `app/src/`.
    </Info>
  </Step>
  
  <Step title="Verify File Location">
    Confirm the file is in the correct location. From the terminal:
    
    ```bash
    ls -la app/google-services.json
    ```
    
    You should see the file listed.
  </Step>
</Steps>

### Understanding google-services.json

The `google-services.json` file contains:

- **Project Information** - Project ID and number
- **Client Information** - App package name and OAuth clients
- **API Keys** - Keys for Firebase services
- **Database URL** - Your Realtime Database endpoint

<Note>
  The Google Services plugin (configured in `build.gradle`) automatically processes this file during build to generate required resources and configurations.
</Note>

## Firebase SDK Configuration

The HuelvaPedia project is already configured with the necessary Firebase dependencies. Here's what's included:

### Project-Level Configuration

In the root `build.gradle:9`:

```gradle build.gradle
classpath 'com.google.gms:google-services:4.3.15'
```

This adds the Google Services plugin to process `google-services.json`.

### Module-Level Configuration

In `app/build.gradle:3`:

```gradle app/build.gradle
plugins {
    id 'com.android.application'
    id 'com.google.gms.google-services'
}
```

This applies the Google Services plugin to the app module.

### Firebase Dependency

In `app/build.gradle:48`:

```gradle app/build.gradle
implementation 'com.google.firebase:firebase-database:19.7.0'
```

This adds the Firebase Realtime Database SDK to the project.

<Tip>
  These configurations are already in place. Once you add `google-services.json`, Android Studio will automatically integrate Firebase during Gradle sync.
</Tip>

## Setting Up Realtime Database

Now create and configure the Firebase Realtime Database.

<Steps>
  <Step title="Navigate to Realtime Database">
    In Firebase Console, go to **Build > Realtime Database** in the left sidebar.
  </Step>
  
  <Step title="Create Database">
    Click **Create Database**.
  </Step>
  
  <Step title="Select Database Location">
    Choose a location close to your users. For a Huelva-focused app, select a European region (e.g., `europe-west1`).
    
    Click **Next**.
  </Step>
  
  <Step title="Configure Security Rules">
    For development, start with **Test mode** to allow read/write access:
    
    ```json
    {
      "rules": {
        ".read": true,
        ".write": true
      }
    }
    ```
    
    <Warning>
      **Test mode is insecure for production.** These rules allow anyone to read and write your data. See [Security Rules](#database-security-rules) below for production-ready rules.
    </Warning>
    
    Click **Enable**.
  </Step>
  
  <Step title="Database Created">
    Your Realtime Database is now created and ready. You'll see the database URL at the top (e.g., `https://your-project.firebaseio.com`).
  </Step>
</Steps>

### Understanding the Data Model

The app uses the `Elementos` class to map Firebase data:

```java Elementos.java (Model Class)
// Each category from Firebase is mapped to an Elementos object
// Properties should include: nombre, descripcion, imagen, etc.
```

The `MainActivity.java:48-70` shows how data is fetched:

```java MainActivity.java
reference = FirebaseDatabase.getInstance().getReference().child("principal");

reference.addValueEventListener(new ValueEventListener()
{
    @Override
    public void onDataChange(@NonNull DataSnapshot snapshot)
    {
        listaElementos.clear();
        for (DataSnapshot dataSnapshot : snapshot.getChildren())
        {
            Elementos elemento = dataSnapshot.getValue(Elementos.class);
            listaElementos.add(elemento);
        }
        adaptador = new Adaptador(MainActivity.this, listaElementos);
        recyclerView.setAdapter(adaptador);
    }

    @Override
    public void onCancelled(@NonNull DatabaseError error)
    {
        Toast.makeText(MainActivity.this, "Algo ha fallado", Toast.LENGTH_SHORT).show();
    }
});
```

<Info>
  The app listens for changes in real-time. Any updates made in the Firebase Console will automatically appear in the app without restarting.
</Info>

## Database Security Rules

For production deployment, implement proper security rules:

### Production-Ready Rules

```json Security Rules
{
  "rules": {
    "principal": {
      ".read": true,
      ".write": "auth != null && auth.uid === 'your-admin-uid'"
    }
  }
}
```

This configuration:
- ✅ Allows **read** access to anyone (public encyclopedia)
- ❌ Restricts **write** access to authenticated admins only

<Tip>
  Since HuelvaPedia is an informational app without user-generated content, public read access is appropriate. Write access should be restricted to content administrators.
</Tip>

### Updating Security Rules

<Steps>
  <Step title="Go to Rules Tab">
    In Firebase Console Realtime Database, click the **Rules** tab
  </Step>
  
  <Step title="Edit Rules">
    Replace the test mode rules with your production rules
  </Step>
  
  <Step title="Publish">
    Click **Publish** to apply the new rules
  </Step>
</Steps>

## Troubleshooting Firebase Issues

### google-services.json Not Found

**Error:** `File google-services.json is missing`

**Solution:**
- Verify the file is in `app/` directory, not `app/src/`
- Check the filename is exactly `google-services.json` (lowercase, no spaces)
- Re-sync Gradle after adding the file

### Package Name Mismatch

**Error:** `No matching client found for package name 'com.example.huelvapedia'`

**Solution:**
- Verify the package name in Firebase Console matches `com.example.huelvapedia`
- Check `applicationId` in `app/build.gradle:11` is correct
- Re-download `google-services.json` if you modified the Firebase app settings

### Database Permission Denied

**Error:** `DatabaseError: Permission denied` in Logcat

**Solution:**
- Check your security rules allow read access
- For testing, use test mode rules (`.read: true`)
- Ensure the database URL is correct in `google-services.json`

### No Data Appears in App

**Problem:** App runs but shows empty screen

**Solution:**
- Verify data exists in Firebase Console under the "principal" node
- Check device/emulator has internet connection
- Look for errors in Logcat related to `FirebaseDatabase`
- Verify database structure matches the expected schema

### Build Fails After Adding google-services.json

**Problem:** Gradle sync or build fails

**Solution:**
- Ensure Google Services plugin is in project `build.gradle:9`
- Verify plugin is applied in `app/build.gradle:3`
- Try **File > Invalidate Caches / Restart** in Android Studio
- Check that Firebase dependency version matches: `firebase-database:19.7.0`
