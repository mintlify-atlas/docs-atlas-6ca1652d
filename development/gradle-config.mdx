## Overview

HuelvaPedia uses Gradle as its build system with Android Gradle Plugin and Google Services integration. This guide covers all build configurations, dependencies, and SDK version requirements defined in `app/build.gradle`.

## Build Configuration

### Plugins

The application uses two essential Gradle plugins:

```groovy app/build.gradle
plugins {
    id 'com.android.application'
    id 'com.google.gms.google-services'
}
```

## Android SDK Configuration

### SDK Versions

HuelvaPedia targets Android devices running API level 27 and above:

```groovy app/build.gradle
android {
    compileSdkVersion 30
    buildToolsVersion "30.0.3"

    defaultConfig {
        applicationId "com.example.huelvapedia"
        minSdkVersion 27
        targetSdkVersion 30
        versionCode 1
        versionName "1.0"

        testInstrumentationRunner "androidx.test.runner.AndroidJUnitRunner"
    }
}
```

<Note>
  **API Level 27** corresponds to Android 8.1 (Oreo), ensuring compatibility with modern Android features while maintaining broad device support.
</Note>

### Version Configuration Breakdown

| Setting | Value | Description |
|---------|-------|-------------|
| `compileSdkVersion` | 30 | Android 11 SDK used for compilation |
| `buildToolsVersion` | 30.0.3 | Build tools version for compilation |
| `minSdkVersion` | 27 | Minimum Android 8.1 (Oreo) required |
| `targetSdkVersion` | 30 | Optimized for Android 11 behavior |
| `versionCode` | 1 | Internal version number |
| `versionName` | "1.0" | User-visible version string |
| `applicationId` | com.example.huelvapedia | Unique app identifier |

## Build Types

### Release Configuration

```groovy app/build.gradle
buildTypes {
    release {
        minifyEnabled false
        proguardFiles getDefaultProguardFile('proguard-android-optimize.txt'), 'proguard-rules.pro'
    }
}
```

<Warning>
  Code minification is currently **disabled** (`minifyEnabled false`). Consider enabling it for production releases to reduce APK size and obfuscate code.
</Warning>

**ProGuard Configuration:**
- Uses default Android optimization rules: `proguard-android-optimize.txt`
- Custom rules defined in: `proguard-rules.pro`

## Upgrading Dependencies

To update dependencies to their latest versions:

<Steps>
  <Step title="Check for Updates">
    Use Android Studio's dependency analyzer:
    ```bash
    ./gradlew dependencyUpdates
    ```
  </Step>
  
  <Step title="Update Version Numbers">
    Modify version numbers in `app/build.gradle` and sync the project.
  </Step>
  
  <Step title="Test Thoroughly">
    Run all tests to ensure compatibility:
    ```bash
    ./gradlew test
    ./gradlew connectedAndroidTest
    ```
  </Step>
</Steps>

<Warning>
  Always test the app thoroughly after updating dependencies, especially major version updates that may introduce breaking changes.
</Warning>
