This guide walks you through the complete process of cloning the HuelvaPedia repository and setting it up in Android Studio. By the end of this guide, you'll have the project ready for development.

<Note>
  Before proceeding, ensure you've completed all [Prerequisites](/development/prerequisites) including installing Android Studio, Android SDK, and Java JDK.
</Note>

## Clone the Repository

First, clone the HuelvaPedia repository from GitHub to your local machine.

<Steps>
  <Step title="Open Terminal or Command Prompt">
    Navigate to the directory where you want to store the project:
    
    ```bash
    cd ~/projects  # or your preferred directory
    ```
  </Step>
  
  <Step title="Clone the Repository">
    Execute the git clone command:
    
    ```bash
    git clone https://github.com/JuancaCalvino/HuelvaPedia.git
    ```
    
    This will create a `HuelvaPedia` directory with all project files.
  </Step>
  
  <Step title="Verify the Clone">
    Navigate into the cloned directory:
    
    ```bash
    cd HuelvaPedia
    ```
    
    List the contents to verify:
    
    ```bash
    ls -la
    ```
    
    You should see files including `build.gradle`, `settings.gradle`, and an `app/` directory.
  </Step>
</Steps>

## Open Project in Android Studio

Now that the repository is cloned, open it in Android Studio.

<Steps>
  <Step title="Launch Android Studio">
    Open Android Studio. If you see the welcome screen, click **Open**. If you have another project open, go to `File > Open...`
  </Step>
  
  <Step title="Navigate to Project Directory">
    Browse to the location where you cloned HuelvaPedia and select the **HuelvaPedia** folder (the root directory containing `build.gradle`).
    
    Click **OK** or **Open** to proceed.
  </Step>
  
  <Step title="Trust the Project">
    Android Studio may ask if you trust the project. Click **Trust Project** to continue.
  </Step>
  
  <Step title="Wait for Indexing">
    Android Studio will begin indexing the project files. This may take a few moments. You'll see progress in the bottom status bar.
  </Step>
</Steps>

<Warning>
  At this stage, the Gradle sync will **fail** because the `google-services.json` file is missing. This is expected and will be resolved in the Firebase Setup step.
</Warning>

## Gradle Sync Process

Once you have the `google-services.json` file in place (covered in the next guide), Android Studio will perform a Gradle sync to download dependencies and configure the build environment.

### Understanding Gradle Sync

Gradle sync is the process where Android Studio:
1. Reads the build configuration files (`build.gradle`)
2. Downloads required dependencies and libraries
3. Configures the project structure
4. Prepares the project for building and running

## Troubleshooting Common Issues

### Missing google-services.json

<Warning>
  **Error:** `File google-services.json is missing. The Google Services Plugin cannot function without it.`
  
  **Solution:** This is expected at this stage. Proceed to the [Firebase Setup](/development/firebase-setup) guide to create and add this file.
</Warning>

### Gradle Sync Timeout

**Problem:** Gradle sync takes too long or times out

**Solutions:**
- Check your internet connection (dependencies must be downloaded)
- Disable VPN if you're using one
- Configure proxy settings if behind a corporate firewall
- Increase Gradle memory in `gradle.properties`:
  ```properties
  org.gradle.jvmargs=-Xmx2048m
  ```

### SDK Version Not Found

**Problem:** Error about missing SDK API Level 30

**Solution:**
1. Open SDK Manager: `Tools > SDK Manager`
2. Under "SDK Platforms", check **Android 11.0 (R)**
3. Under "SDK Tools", check **Android SDK Build-Tools 30.0.3**
4. Click "Apply" to download and install

### Build Tools Version Conflict

**Problem:** Build tools version mismatch errors

**Solution:** The project requires Build Tools 30.0.3:
```gradle app/build.gradle
buildToolsVersion "30.0.3"
```

Install it via SDK Manager if missing.

## Verifying the Installation

Once you have `google-services.json` in place and Gradle sync completes successfully:

<Steps>
  <Step title="Check Build Output">
    Look at the "Build" tab at the bottom of Android Studio. You should see "BUILD SUCCESSFUL" with no errors.
  </Step>
  
  <Step title="Verify Project Structure">
    In the Project view (left sidebar), ensure all folders are properly recognized:
    - `java` folder should be marked in green (source root)
    - `res` folder should have the proper resource structure
    - No red underlines in `build.gradle` files
  </Step>
  
  <Step title="Check Dependencies">
    Open `app/build.gradle` and verify that all dependencies are resolved (no red underlines)
  </Step>
</Steps>

## Code Example: Main Activity Connection

Here's how the app connects to Firebase in the main activity (from `MainActivity.java:48`):

```java MainActivity.java
reference = FirebaseDatabase.getInstance().getReference().child("principal");

reference.addValueEventListener(new ValueEventListener()
{
    @Override
    public void onDataChange(@NonNull DataSnapshot snapshot)
    {
        listaElementos.clear();
        for (DataSnapshot dataSnapshot : snapshot.getChildren())
        {
            Elementos elemento = dataSnapshot.getValue(Elementos.class);
            listaElementos.add(elemento);
        }
        adaptador = new Adaptador(MainActivity.this, listaElementos);
        recyclerView.setAdapter(adaptador);
    }

    @Override
    public void onCancelled(@NonNull DatabaseError error)
    {
        Toast.makeText(MainActivity.this, "Algo ha fallado", Toast.LENGTH_SHORT).show();
    }
});
```

This code demonstrates why Firebase configuration is essential - the app queries the "principal" node to populate the main category list.
