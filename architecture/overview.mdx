## Introduction

HuelvaPedia follows a traditional Android architecture pattern built around Activities, RecyclerViews, and Firebase Realtime Database. The app uses a hierarchical navigation structure with a splash screen, main listing, and detailed views.

## Application Flow

The app follows a linear navigation flow with hierarchical data browsing:

<Steps>
  <Step title="Splash Screen">
    `PantallaCarga` checks network connectivity and displays animated branding before proceeding
  </Step>
  <Step title="Main Categories">
    `MainActivity` displays the primary categories (monuments, beaches, restaurants, etc.) from Firebase
  </Step>
  <Step title="Subcategories">
    `Secundaria` shows items within a selected category, fetched dynamically from Firebase
  </Step>
  <Step title="Weather Feature">
    `Tiempo` provides current weather information for Huelva via OpenWeatherMap API
  </Step>
</Steps>

### MainActivity (Primary Categories)

The main landing screen displaying top-level categories from Firebase.

```java MainActivity.java:32-76
@Override
protected void onCreate(Bundle savedInstanceState)
{
    super.onCreate(savedInstanceState);
    setContentView(R.layout.activity_main);

    botonTiempo = findViewById(R.id.botontiempo);

    ActionBar actionBar = getSupportActionBar();
    actionBar.setIcon(R.mipmap.ic_escudohuelva);
    actionBar.setDisplayShowHomeEnabled(true);

    recyclerView = findViewById(R.id.recicler);
    recyclerView.setLayoutManager(new LinearLayoutManager(this));

    listaElementos = new ArrayList<>();

    reference = FirebaseDatabase.getInstance().getReference().child("principal");

    reference.addValueEventListener(new ValueEventListener()
    {
        @Override
        public void onDataChange(@NonNull DataSnapshot snapshot)
        {
            listaElementos.clear(); // Evita duplicación de datos cuando Firebase notifica cambios
            for (DataSnapshot dataSnapshot : snapshot.getChildren())
            {
                Elementos elemento = dataSnapshot.getValue(Elementos.class);
                listaElementos.add(elemento);
            }
            adaptador = new Adaptador(MainActivity.this, listaElementos);
            recyclerView.setAdapter(adaptador);
        }

        @Override
        public void onCancelled(@NonNull DatabaseError error)
        {
            Toast.makeText(MainActivity.this, "Algo ha fallado", Toast.LENGTH_SHORT).show();
        }
    });

    botonTiempo.setOnClickListener(v -> {
        Intent intent = new Intent(v.getContext(), Tiempo.class);
        startActivity(intent);
    });
}
```

**Architecture Components:**
- **RecyclerView** with `LinearLayoutManager` for efficient scrolling
- **Firebase ValueEventListener** for real-time data synchronization
- **Custom Adapter** (`Adaptador`) to bind data to views
- **ActionBar** customization with Huelva coat of arms icon
- **Weather Button** for navigating to weather information

### Secundaria (Subcategory View)

Displays items within a selected category, sharing the same layout as MainActivity but with dynamic Firebase references.

```java Secundaria.java:32-78

{
    super.onCreate(savedInstanceState);
    setContentView(R.layout.activity_main);

    botonTiempo = findViewById(R.id.botontiempo);

    String nombreReferencia = intent.getStringExtra("Nombre")
            .replace(" ", "")
            .toLowerCase();

    reference = FirebaseDatabase.getInstance().getReference().child(nombreReferencia);

    reference.addValueEventListener(new ValueEventListener()
    {
        @Override
        public void onDataChange(@NonNull DataSnapshot snapshot)
        {
            listaElementos.clear(); // Evita duplicación de datos cuando Firebase notifica cambios
            for (DataSnapshot dataSnapshot : snapshot.getChildren())
            {
                Elementos elemento = dataSnapshot.getValue(Elementos.class);
                listaElementos.add(elemento);
            }
            adaptador = new Adaptador(Secundaria.this, listaElementos);
            recyclerView.setAdapter(adaptAdapter);
        }

        @Override
        public void onCancelled(@NonNull DatabaseError error)
        {
            Toast.makeText(Secundaria.this, "Algo ha fallado", Toast.LENGTH_SHORT).show();
        }
    });
}
```

**Key Differences from MainActivity:**
- Receives category name via Intent extras
- Normalizes category name (removes spaces, converts to lowercase) for Firebase path
- Hides weather button for cleaner subcategory navigation
- Uses same layout (`activity_main.xml`) for consistency

### Tiempo (Weather Activity)

Fetches and displays current weather data for Huelva using OpenWeatherMap API.

```java Tiempo.java:35-92
@Override
protected void onCreate(Bundle savedInstanceState) {
    super.onCreate(savedInstanceState);
    setContentView(R.layout.activity_tiempo);

    tvTemperatura = findViewById(R.id.temperatura);
    tvDescripcion = findViewById(R.id.descripcion);
    tvSensacion = findViewById(R.id.sensacion);
    tvPresion = findViewById(R.id.presion);
    tvHumedad = findViewById(R.id.humedad);
    tvNubes = findViewById(R.id.nubes);
    tvViento = findViewById(R.id.viento);

    imagenDelTiempo = findViewById(R.id.imagentiempo);

    ActionBar actionBar = getSupportActionBar();
    actionBar.setIcon(R.mipmap.ic_escudohuelva);
    actionBar.setDisplayShowHomeEnabled(true);

    String urlCompleta = URL_BASE + "?q=Huelva,ES&appid=" + API_KEY;

    StringRequest stringRequest = new StringRequest(Request.Method.POST, urlCompleta,
            response -> {
                try
                {
                    JSONObject jsonResponse = new JSONObject(response);
                    JSONArray jsonArray = jsonResponse.getJSONArray("weather");
                    JSONObject jsonObjectWeather = jsonArray.getJSONObject(0);
                    String descripcion = jsonObjectWeather.getString("description");
                    JSONObject jsonObjectMain = jsonResponse.getJSONObject("main");
                    double temp = jsonObjectMain.getDouble("temp") - 273.15;
                    double sensacion = jsonObjectMain.getDouble("feels_like") - 273.15;
                    float presion = jsonObjectMain.getInt("pressure");
                    int humedad = jsonObjectMain.getInt("humidity");
                    JSONObject jsonObjectWind = jsonResponse.getJSONObject("wind");
                    String viento = jsonObjectWind.getString("speed");
                    JSONObject jsonObjectClouds = jsonResponse.getJSONObject("clouds");
                    String nubes = jsonObjectClouds.getString("all");

                    traducirDescripcion(descripcion);

                    tvTemperatura.setText(formatoDecimal.format(temp) + "ºC");
                    tvSensacion.setText("Sensacion térmica: " + formatoDecimal.format(sensacion) + "ºC");
                    tvHumedad.setText("Humedad: " + humedad + "%");
                    tvNubes.setText("Cantidad de nubes: " + nubes + "%");
                    tvViento.setText("Velocidad del viento: " + viento + "m/s (metros por segundo)");
                    tvPresion.setText("Presión atmosférica: " + presion + " hPa");
                }
                catch (JSONException e)
                {
                    e.printStackTrace();
                }
            },
            error -> Toast.makeText(getApplicationContext(), error.toString().trim(), Toast.LENGTH_SHORT).show()
    );

    RequestQueue requestQueue = Volley.newRequestQueue(getApplicationContext());
    requestQueue.add(stringRequest);
}
```

**Features:**
- Uses **Volley** library for HTTP networking
- Parses JSON weather data from OpenWeatherMap
- Converts Kelvin temperatures to Celsius
- Translates English descriptions to Spanish
- Sets weather icons based on conditions

<Warning>
  The API key is currently hardcoded in `Tiempo.java:27`. Consider moving it to `local.properties` and accessing via `BuildConfig` to prevent exposure in version control.
</Warning>

The adapter intelligently shows/hides action buttons based on the `ultimo` flag:

<Info>
  Items marked as **not last** (`ultimo = false`) navigate to `Secundaria` to show subcategories. Items marked as **last** (`ultimo = true`) display action buttons (map, call, web link) instead.
</Info>

## Firebase Integration

### Database Structure

HuelvaPedia uses Firebase Realtime Database with a hierarchical structure:

```
root/
├── principal/
│   ├── item1 (Elementos)
│   ├── item2 (Elementos)
│   └── ...
├── playas/
│   ├── beach1 (Elementos)
│   └── beach2 (Elementos)
├── monumentos/
│   ├── monument1 (Elementos)
│   └── monument2 (Elementos)
└── [other categories]/
```

### Real-time Synchronization

Both `MainActivity` and `Secundaria` use `ValueEventListener` for real-time updates:

```java MainActivity.java:50-70
reference.addValueEventListener(new ValueEventListener()
{
    @Override
    public void onDataChange(@NonNull DataSnapshot snapshot)
    {
        listaElementos.clear(); // Evita duplicación de datos cuando Firebase notifica cambios
        for (DataSnapshot dataSnapshot : snapshot.getChildren())
        {
            Elementos elemento = dataSnapshot.getValue(Elementos.class);
            listaElementos.add(elemento);
        }
        adaptador = new Adaptador(MainActivity.this, listaElementos);
        recyclerView.setAdapter(adaptador);
    }

    @Override
    public void onCancelled(@NonNull DatabaseError error)
    {
        Toast.makeText(MainActivity.this, "Algo ha fallado", Toast.LENGTH_SHORT).show();
    }
});
```

**Benefits:**
- Automatic UI updates when Firebase data changes
- No manual refresh required
- Clear operation prevents duplicate entries on data change events

### Data Deserialization

Firebase automatically deserializes JSON data into `Elementos` objects:

```java
Elementos elemento = dataSnapshot.getValue(Elementos.class);
```

This requires the `Elementos` class to have:
- An empty constructor
- Getter/setter methods matching Firebase field names

## Performance Considerations

### RecyclerView Optimization

- **LinearLayoutManager** provides smooth vertical scrolling
- **ViewHolder pattern** minimizes findViewById() calls
- **Item view recycling** reduces memory allocations

### Image Loading

Picasso handles image optimization:
- Downloads images asynchronously (no UI blocking)
- Caches images in memory and disk
- Automatically scales images to fit ImageView dimensions

### Firebase Efficiency

- **ValueEventListener** receives only data changes (not full re-syncs)
- Data automatically serialized/deserialized into Java objects
- Firebase SDK handles connection pooling and retry logic

## Security Considerations

<Warning>
  **API Key Exposure**: The OpenWeatherMap API key is currently hardcoded in `Tiempo.java`. Move sensitive keys to `local.properties` or use Android's BuildConfig to keep them out of version control.
</Warning>

<Info>
  **Firebase Security Rules**: Ensure Firebase Realtime Database rules properly restrict read/write access. Since this is a read-only encyclopedia app, Firebase rules should allow public reads but restrict writes to authenticated admin users only.
</Info>
