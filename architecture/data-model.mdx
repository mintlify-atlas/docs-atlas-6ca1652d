## Overview

HuelvaPedia's data model centers around the **Elementos** class, which represents individual encyclopedia entries (monuments, beaches, restaurants, etc.). This class serves as both a Plain Old Java Object (POJO) and a Firebase-compatible data transfer object.

### Field Reference

<Tabs>
  <Tab title="Core Fields">
    | Field | Type | Description | Example |
    |-------|------|-------------|----------|
    | `nombre` | `String` | Display name of the entry | `"Catedral de Huelva"` |
    | `foto` | `String` | URL to the entry's image | `"https://firebase.../image.jpg"` |
    | `descripcion` | `String` | Brief description text | `"Historic 16th century cathedral..."` |
  </Tab>
  <Tab title="Location Fields">

    |-------|------|-------------|----------|
    | `ubicacion1` | `String` | Map URL base (optional) | `"geo:0,0?q="` |

      **Combined Usage**: `ubicacion1` and `ubicacion2` are concatenated to form a complete map URL. If both are `null`, the location button is hidden.
    </Info>

  <Tab title="Action Fields">
    | Field | Type | Description | Example |
    |-------|------|-------------|----------|

      The `ultimo` field controls navigation behavior:

## Firebase Database Schema

### Hierarchical Structure

HuelvaPedia uses a two-level hierarchy in Firebase Realtime Database:

```
root/
├── principal/              # Top-level categories
│   ├── item-001
│   │   ├── nombre: "Playas"
│   │   ├── foto: "https://..."
│   │   ├── descripcion: "Beautiful beaches..."
│   │   ├── ubicacion1: null
│   │   ├── ubicacion2: null
│   │   ├── telefono: null
│   │   ├── ultimo: false
│   │   └── enlace: null
│   ├── item-002
│   │   ├── nombre: "Monumentos"
│   │   └── ...
│   └── item-003
│       ├── nombre: "Restaurantes"
│       └── ...
│
├── playas/                 # Subcategory (normalized from "Playas")
│   ├── beach-001
│   │   ├── nombre: "Playa de Punta Umbría"
│   │   ├── foto: "https://..."
│   │   ├── descripcion: "Golden sand beach..."
│   │   ├── ubicacion1: "geo:0,0?q="
│   │   ├── ubicacion2: "Punta Umbría, Huelva"
│   │   ├── telefono: "+34 959 123 456"
│   │   ├── ultimo: true
│   │   └── enlace: "https://..."
│   └── beach-002
│       ├── nombre: "Playa de Mazagón"
│       └── ...
│
├── monumentos/             # Subcategory (normalized from "Monumentos")
│   ├── monument-001
│   │   ├── nombre: "Catedral de la Merced"
│   │   └── ...
│   └── monument-002
│       └── ...
│
└── [other categories]/     # Additional normalized categories
    └── ...
```

**Examples:**

## Data Usage Patterns

### Pattern 1: Category Entries (ultimo = false)

Category entries link to subcategories and don't display action buttons.

**Firebase Data:**
```json
{
  "nombre": "Playas",
  "foto": "https://.../beaches-category.jpg",
  "descripcion": "Discover Huelva's beautiful Atlantic beaches",
  "ubicacion1": null,
  "ubicacion2": null,
  "telefono": null,
  "ultimo": false,
  "enlace": null
}
```

**UI Behavior:**
- Displays name, photo, and description
- Hides all action buttons
- Entire item is clickable
- Navigates to `Secundaria` with "playas" reference

**Adapter Code:**
```java Adaptador.java:54-67
if (!elemento.getUltimo()) {
    holder.botonUbicacion.setVisibility(View.GONE);
    holder.botonLlamar.setVisibility(View.GONE);
    holder.botonEnlace.setVisibility(View.GONE);

    if (!tieneUbicacion && !tieneTelefono && !tieneEnlace)
    {
        holder.itemView.setOnClickListener(v -> {
            Intent intent = new Intent(v.getContext(), Secundaria.class);
            intent.putExtra("Nombre", elemento.getNombre());
            v.getContext().startActivity(intent);
        });
    }
}
```

### Pattern 2: Terminal Entries (ultimo = true)

Terminal entries represent actual encyclopedia items with actionable information.

**Firebase Data:**
```json
{
  "nombre": "Playa de Mazagón",
  "foto": "https://.../mazagon-beach.jpg",
  "descripcion": "Expansive golden sand beach with clear waters and excellent facilities",
  "ubicacion1": "geo:0,0?q=",
  "ubicacion2": "Playa de Mazagón, 21130 Mazagón, Huelva",
  "telefono": "+34 959 376 200",
  "ultimo": true,
  "enlace": "https://turismohuelva.org/mazagon"
}
```

**UI Behavior:**
- Displays name, photo, and description
- Shows **"Ver ubicación"** button (opens map with location)
- Shows **"Llamar"** button (opens dialer with phone number)
- Shows **"Ver fauna"** button (opens web link)
- Item is NOT clickable for navigation

**Adapter Code:**
```java Adaptador.java:69-113
else // Es el último elemento, solo permite pulsar botones de acción
{
    // Botón de ubicación (mapa)
    if (tieneUbicacion) {
        holder.botonUbicacion.setOnClickListener(v -> {
            Intent intent = new Intent(Intent.ACTION_VIEW);
            intent.setData(Uri.parse(elemento.getUbicacion1() + Uri.encode(elemento.getUbicacion2())));
            Intent chooser = Intent.createChooser(intent, "Launch Maps");
            v.getContext().startActivity(chooser);
        });
    } else {
        holder.botonUbicacion.setVisibility(View.GONE);
    }

    // Botón de teléfono (llamar)
    if (tieneTelefono) {
        holder.botonLlamar.setOnClickListener(v -> {
            Intent intent = new Intent(Intent.ACTION_DIAL);
            intent.setData(Uri.parse("tel:" + elemento.getTelefono()));
            v.getContext().startActivity(intent);
        });
    } else {
        holder.botonLlamar.setVisibility(View.GONE);
    }

    // Botón de enlace (web)
    if (tieneEnlace) {
        holder.botonEnlace.setOnClickListener(v -> {
            Uri link = Uri.parse(elemento.getEnlace());
            Intent intent = new Intent(Intent.ACTION_VIEW, link);
            v.getContext().startActivity(intent);
        });
    } else {
        holder.botonEnlace.setVisibility(View.GONE);
    }
}
```

### Recommended Phone Formats

| Format | Example | Description |
|--------|---------|-------------|
| **International** | `+34 959 123 456` | Preferred format with country code |
| **National** | `959 123 456` | Works within Spain |
| **With Parentheses** | `+34 (959) 123-456` | Also supported |

<Info>
  **ACTION_DIAL vs ACTION_CALL**: The app uses `ACTION_DIAL`, which opens the dialer with the number pre-filled but doesn't automatically initiate the call. This avoids requiring the `CALL_PHONE` permission.
</Info>

### Validation Rules

<Steps>
  <Step title="Name Validation">
    - Must not be empty
    - Used for display and Firebase reference normalization
    - Avoid special characters that might break normalization
  </Step>
  <Step title="Photo URL Validation">
    - Must be a valid HTTP/HTTPS URL
    - Image should be accessible without authentication (or use Firebase Storage tokens)
    - Recommended image dimensions: 800x600 to 1200x900
  </Step>
  <Step title="Location Validation">
    - If `ubicacion1` is set, `ubicacion2` must also be set
    - Together they must form a valid `geo:` URI
  </Step>
  <Step title="Boolean Validation">
    - `ultimo` must be explicitly `true` or `false` (not null)
    - Controls entire navigation flow
  </Step>
</Steps>

## Performance Optimization

### Data Loading Strategy

- **Lazy Loading**: Firebase only loads the requested path (`principal` or specific category)
- **Incremental Updates**: ValueEventListener receives only changed data after initial load
- **Local Caching**: Firebase SDK caches data locally for offline access

### Memory Management

```java
listaElementos.clear(); // Clear list before repopulating
```

This prevents memory leaks by ensuring old `Elementos` objects are garbage collected when Firebase sends updates.

### Image Caching

Picasso automatically caches images:
- **Memory cache**: Recently viewed images stay in RAM
- **Disk cache**: Downloaded images persist across app restarts
- **Automatic eviction**: Caches are cleared based on LRU (Least Recently Used) policy
