## Installation

<Steps>
  <Step title="Clone the Repository">
    Get the HuelvaPedia source code from GitHub:

    <CodeGroup>
      ```bash HTTPS
      git clone https://github.com/JuancaCalvino/HuelvaPedia.git
      ```

      ```bash SSH
      git clone git@github.com:JuancaCalvino/HuelvaPedia.git
      ```

      ```bash GitHub CLI
      gh repo clone JuancaCalvino/HuelvaPedia
      ```
    </CodeGroup>

    Navigate to the project directory:
    ```bash
    cd HuelvaPedia
    ```
  </Step>

  <Step title="Open in Android Studio">
    1. Launch Android Studio
    2. Select **File > Open...** from the menu
    3. Navigate to the cloned `HuelvaPedia` folder
    4. Click **OK** to open the project
    5. Wait for Gradle to sync (this may take a few minutes on first load)

    <Tip>
      If you encounter Gradle sync issues, try **File > Invalidate Caches and Restart**
    </Tip>
  </Step>

  <Step title="Configure Firebase">
    <Warning>
      The `google-services.json` file is **NOT included** in the repository for security reasons. You must set up your own Firebase project.
    </Warning>

    ### Create Firebase Project

    1. Go to the [Firebase Console](https://console.firebase.google.com/)
    2. Click **Add Project** or select an existing project
    3. Follow the setup wizard to create your project

    ### Add Android App to Firebase

    1. In the Firebase Console, click **Add app** and select Android
    2. Enter the package name: `com.example.huelvapedia`
    3. (Optional) Add an app nickname: "HuelvaPedia"
    4. Click **Register app**

    ### Download Configuration File

    1. Download the `google-services.json` file
    2. Place it in the `app/` directory of your project:
    ```
    HuelvaPedia/
    └── app/
        ├── build.gradle
        ├── google-services.json  ← Place here
        └── src/
    ```

    <Info>
      The app/build.gradle file already includes the Google Services plugin:
      ```gradle
      plugins {
          id 'com.android.application'
          id 'com.google.gms.google-services'
      }
      ```
    </Info>
  </Step>

  <Step title="Set Up Firebase Realtime Database">
    HuelvaPedia uses Firebase Realtime Database to store all content dynamically.

    ### Enable Realtime Database

    1. In Firebase Console, navigate to **Build > Realtime Database**
    2. Click **Create Database**
    3. Choose a location (preferably close to your users)
    4. Start in **Test mode** for development (remember to secure it later)

    ### Database Structure

    The app expects data under a `principal` node. Here's the structure:

    ```json
    {
      "principal": {
        "item1": {
          "nombre": "Catedral de Huelva",
          "foto": "https://example.com/image.jpg",
          "descripcion": "Historic cathedral in the heart of Huelva",
          "ubicacion1": "Plaza de la Merced",
          "ubicacion2": "Huelva, Spain",
          "telefono": "+34 959 24 61 64",
          "ultimo": false,
          "enlace": "https://example.com/more-info"
        },
        "item2": {
          "nombre": "Monumento a Colón",
          "foto": "https://example.com/colon.jpg",
          "descripcion": "Monument dedicated to Christopher Columbus",
          "ubicacion1": "Punta del Sebo",
          "ubicacion2": "Huelva, Spain",
          "telefono": "",
          "ultimo": false,
          "enlace": "https://example.com/colon"
        }
      }
    }
    ```

    <Tip>
      The `Elementos` class (app/src/main/java/com/example/huelvapedia/Elementos.java:3) defines the data model with fields for nombre, foto, descripcion, ubicacion1, ubicacion2, telefono, ultimo, and enlace.
    </Tip>

    ### Configure Database Rules

    For development, you can use these rules (in Firebase Console > Rules tab):
    ```json
    {
      "rules": {
        ".read": true,
        ".write": false
      }
    }
    ```

    <Warning>
      These rules allow public read access. Adjust for production based on your security requirements.
    </Warning>
  </Step>

  <Step title="Build and Run">
    ### Sync Gradle

    If not already synced, click the **Sync Project with Gradle Files** button in the toolbar, or:
    - Go to **File > Sync Project with Gradle Files**

    ### Run the App

    <Tabs>
      <Tab title="Physical Device">
        1. Enable **Developer Options** on your Android device
        2. Enable **USB Debugging**
        3. Connect your device via USB
        4. Click the **Run** button (▶) or press `Shift + F10`
        5. Select your device from the list
      </Tab>

      <Tab title="Emulator">
        1. Open **AVD Manager** (Tools > Device Manager)
        2. Create a new virtual device or select an existing one
        3. Ensure the device runs API 27 or higher
        4. Click the **Run** button (▶) or press `Shift + F10`
        5. Select your emulator from the list
      </Tab>
    </Tabs>

    <Info>
      **Internet connection is required** - The app fetches all content from Firebase in real-time.
    </Info>
  </Step>

  <Step title="Verify Installation">
    Once the app launches, you should see:

    1. **Splash Screen** (PantallaCarga activity)
    2. **Main Activity** with a RecyclerView displaying categories from Firebase
    3. Categories loaded from the `principal` node in your database

    If you see "Algo ha fallado" (Something failed), check:
    - Firebase configuration is correct
    - `google-services.json` is in the right location
    - Database has data under the `principal` node
    - Internet connection is active
    - Database rules allow read access

    <Tip>
      Check Android Studio's Logcat for detailed error messages if something goes wrong.
    </Tip>
  </Step>
</Steps>

### Firebase Integration

The app connects to Firebase in `MainActivity.java:48`:

```java
reference = FirebaseDatabase.getInstance().getReference().child("principal");

reference.addValueEventListener(new ValueEventListener() {
    @Override
    public void onDataChange(@NonNull DataSnapshot snapshot) {
        listaElementos.clear();
        for (DataSnapshot dataSnapshot : snapshot.getChildren()) {
            Elementos elemento = dataSnapshot.getValue(Elementos.class);
            listaElementos.add(elemento);
        }
        adaptador = new Adaptador(MainActivity.this, listaElementos);
        recyclerView.setAdapter(adaptador);
    }

    @Override
    public void onCancelled(@NonNull DatabaseError error) {
        Toast.makeText(MainActivity.this, "Algo ha fallado", Toast.LENGTH_SHORT).show();
    }
});
```

## Troubleshooting

<AccordionGroup>
  <Accordion title="Gradle sync fails">
    **Solution:**
    - Check your internet connection
    - Try **File > Invalidate Caches and Restart**
    - Ensure Java JDK 11+ is installed
    - Update Android Studio to the latest version
  </Accordion>

  <Accordion title="App crashes on launch">
    **Solution:**
    - Verify `google-services.json` is in the `app/` directory
    - Check Firebase project configuration matches package name
    - Review Logcat for specific error messages
    - Ensure minimum SDK version is met (API 27+)
  </Accordion>

  <Accordion title="No data appears in the app">
    **Solution:**
    - Verify Firebase Realtime Database has data under `principal` node
    - Check database rules allow read access
    - Confirm device/emulator has internet connectivity
    - Check Logcat for Firebase connection errors
  </Accordion>

  <Accordion title="Google Services plugin error">
    **Solution:**
    - Ensure `google-services.json` is present
    - Verify the file is valid JSON
    - Check that the package name in Firebase matches `com.example.huelvapedia`
    - Clean and rebuild project (Build > Clean Project, then Build > Rebuild Project)
  </Accordion>
</AccordionGroup>
