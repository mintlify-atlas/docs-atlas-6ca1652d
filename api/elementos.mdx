## Overview

The `Elementos` class is the primary data model for HuelvaPedia entries. It stores all information about encyclopedia items including text content, images, location data, contact information, and navigation metadata.

<Note>
This class follows the JavaBean pattern with private fields and public getter/setter methods for data encapsulation.
</Note>

## Fields

All fields are private and accessed through getter/setter methods:

| Field | Type | Description |
|-------|------|-------------|
| `nombre` | `String` | The name or title of the encyclopedia entry |
| `foto` | `String` | URL or path to the entry's image |
| `descripcion` | `String` | Descriptive text about the entry |
| `ubicacion1` | `String` | First part of the location URL (base maps URL) |
| `ubicacion2` | `String` | Second part of the location URL (encoded address) |
| `telefono` | `String` | Contact phone number for the entry |
| `ultimo` | `Boolean` | Flag indicating if this is a terminal/leaf entry (no sub-items) |
| `enlace` | `String` | External web link for additional information |

## Constructors

### Default Constructor

```java
public Elementos()
```

Creates an empty `Elementos` object with all fields set to `null`.

**Usage Example:**
```java
Elementos elemento = new Elementos();
elemento.setNombre("Plaza de las Monjas");
elemento.setDescripcion("Historic square in central Huelva");
```

### Full Constructor

```java
public Elementos(String nombre, String foto, String descripcion, 
                 String ubicacion1, String ubicacion2, String telefono, 
                 Boolean ultimo, String enlace)
```

Creates an `Elementos` object with all fields initialized.

**Parameters:**
- `nombre` - The entry name
- `foto` - Image URL
- `descripcion` - Entry description
- `ubicacion1` - Maps URL base
- `ubicacion2` - Location address
- `telefono` - Contact phone number
- `ultimo` - Whether this is a terminal entry
- `enlace` - External reference link

**Usage Example:**
```java
Elementos monumento = new Elementos(
    "Catedral de Huelva",
    "https://example.com/catedral.jpg",
    "Catedral de estilo barroco del siglo XVIII",
    "geo:0,0?q=",
    "Plaza de la Merced, Huelva",
    "+34959240512",
    true,
    "https://es.wikipedia.org/wiki/Catedral_de_Huelva"
);
```

## Usage Patterns

### Creating Entries with Optional Fields

```java
// Entry without location or contact info
Elementos historia = new Elementos();
historia.setNombre("Historia de Huelva");
historia.setDescripcion("Descubre el pasado de la ciudad");
historia.setFoto("https://example.com/historia.jpg");
historia.setUltimo(false); // Has sub-entries

// Terminal entry with full contact info
Elementos museo = new Elementos();
museo.setNombre("Museo de Huelva");
museo.setDescripcion("Museo provincial de arte y arqueología");
museo.setFoto("https://example.com/museo.jpg");
museo.setUbicacion1("geo:0,0?q=");
museo.setUbicacion2("Alameda Sundheim, 13, Huelva");
museo.setTelefono("+34959650424");
museo.setEnlace("https://www.museosdeandalucia.es/web/museodehuelva");
museo.setUltimo(true); // No sub-entries
```

## Field Usage Guidelines

<CardGroup cols={2}>
<Card title="Location Fields" icon="map-marker">
Both `ubicacion1` and `ubicacion2` should be set together. They combine to form a complete maps URI that can be opened in any maps application.
</Card>

<Card title="Ultimo Flag" icon="flag">
Set to `true` for terminal entries that display action buttons (map, call, web). Set to `false` for category entries that navigate to sub-items.
</Card>

<Card title="Optional Fields" icon="circle-question">
`telefono`, `ubicacion1/2`, and `enlace` can be `null`. The UI will automatically hide corresponding action buttons.
</Card>

<Card title="Required Fields" icon="circle-check">
`nombre`, `foto`, and `descripcion` should always be populated for proper display in the RecyclerView.
</Card>
</CardGroup>
